# 前言


## 由何而起？

我在2023年初开始的一年间，在Covid-19解封之后，大学一年级下学期开始 我主要作为产品和测试，参加了十几场黑客松。

（如 Ethereum Beijing，Singapore，Hangzhou 、Polkadot Winter Hong kong、Circle Hackathon etc.）


于是我身边不少听过我的经历，但没有参与过黑客松又对黑客松很感兴趣的朋友 经常问我：

**如何去开启第一场黑客松？** & **我不会开发/不是hacker，值得去参加一次吗？**

---

不少朋友也是因为各种原因而无法参加相关领域的黑客松————害怕自己做不出什么东西、感觉自己能力有限、找不到最近的黑客松活动信息......

基于上述情况，我创建了这个仓库，旨在帮助所有的黑客松小白**能开启他人生中的第一场黑客松**。

## 为什么你要去打黑客松？

我认为每一位

- 刚开始了解一个领域（不止是Web3）
- 学习一门新语言，无论何种技术栈
- 会一些技能的朋友，例如设计，演讲，哪怕是做PPT

**都可以去尝试去体验黑客松的整个赛程**

<br>

无论是线上或者是线下 ***（当然线下氛围更好）***

无论你是开发者，产品，想要让自己的想法变成现实，做一些自己的产品，

还是设计师，“布道者”，让团队的作品让更多人看到其中的创新点或闪光点。

---

特别是对于所有朋友来说，黑客松是很好的一次获得自我提升以及接触人脉，认识新朋友的机会。

特别是提高自己的技术实现能力和商业计划水平。

我[自己](http://github.com/yanboishere)深有体会。

## 黑客松分类

黑客松的类型各有不同，既有大厂（Microsoft），公链（ [以太坊](https://ethereum.org/zh/community/events/)、[Solana](https://solana.com/events)、[Sui](https://sui.io/community-events-hub) ，也有预言机、跨链桥（Chainlink、Circle CCTP）等各类产品，当然也有社区和组织开展的开放性主题黑客松（如[AdventureX](https://adventure-x.org/) ）

第一次打黑客松，建议最好是去线下去体验，感受更浓厚的氛围和更好的交流体验。

## 比赛过程

### 前期报名/组队
#### 报名
在前期报名的过程中，主办方会在活动官网发布报名的表格（也有直接提交项目的黑客松）

来去通过收集你的个人信息 

可能包括但不限于以下信息：

- 个人基本信息
  - 姓名
  - 年龄
  - 联系方式
    - 邮件
    - 微信
    - Telegram
    - Discord
  - 个人社交媒体
    - X
  - 作品集
    - GitHub
    - 博客
- 个人经验
  - 本行业从业年限
  - 曾经做过什么项目
  - 是否是首次参赛
- 在团队中的角色
  - 技术
    - 前端
    - 后端
    - 智能合约
  - 产品
  - 设计
    - UI
    - UX
- 是否为学生
- 在本次黑客松想做的事情（项目）

---

提交完以上信息 请等待项目方给你的邮件或者社交媒体回复↩️ 

- 是否成功报名
- 有名额限制的热门黑客松
  - 是否入选到黑客松中
 
#### 组队
在收到/未收到 报名及入选成功信息后（最好是在收到报名及入选成功信息后）

如果你不想自己去solo独立搞事情，最好去黑客松官方的社群，或者项目初期展示时，去找到能力适合自己项目、可以并肩作战的队友。

（黑客松官方可能会组织未组队的朋友/正在急召队友的队伍 在线上会议或者通过表格信息的形式自由组织组队）
 
### 项目初期展示

有部分线下/线上黑客松，会在开赛之初 让所有参赛者用1-3分钟时间轮流介绍自己的项目。

此时需要你简短的去概括自己的项目/介绍自己的团队，

最好讲述的让场下的朋友们印象深刻些，讲出自己的创新点或项目特点。

这时候也可以提到自己的队伍缺少什么角色的队友，如前端，产品（一般情况下是缺少技术角色的队友），问下在场的朋友有没有还未组队的情况。

## Q&A 答疑
### Q：我不会技术 可不可以参加黑客松？

A：
借用Jupiter老师的总结

其实不会技术的同学也可以参加黑客松，黑客松虽然关注技术，但是其实也有很多东西可以关注。

比如如果做B端产品，那其实同时参加黑客松的这些项目就可能是你的客户，就需要你去BD（Bussiness Developer）。

另外也可以在这个过程，比如pitch或者demo day的时候去锻炼讲故事的能力，讲好一个故事对于任何产品来说都很重要，任何产品都需要有一定技术的人来叙事的，所以这个能力真的非常需要关注，不能只盯着技术。

黑客松还会涌现出一些趋势性的或者很新的产品，可以去做一些相关赛道的分析，在黑客松之外抓住机会。

---

包括我自己大一刚开始的时候也是技术小白的情况下参与了人生第一次黑客松，还作为队长带领团队去做产品。

虽然那时候不懂技术，开发对我来说约等于完全不会，但很感谢的是我找到了一些很会帮我解决技术和开发问题的朋友。

特别是在那次黑客松上，我发现大家使用各种AIGC（人工智能生成内容）工具去做产品中的代码、美术（LOGO、PPT、海报）部分，我认为未来借助于AIGC应用，一人去参赛或者一人公司的情况也会越来越多。

## Q：黑客松是否只能在比赛期间构建项目？
A：理论上是这样的 包括很多黑客松的规定是只能在黑客松期间去构建自己的项目 

但实际上仍有很多项目是在黑客松之前的很长一段时间已经构建很久周期去做的项目再去进行提交

个人实际上是反对这种做法的 但是如果你是想在赛前去 提前做测试、完善想法和思路、了解需要的技术栈情况

或者是在之前项目的基础上 在这次黑客松赛程期间 加入新的功能/做一些新的尝试 

那是在可允许范围之内的
